毒舌锐评

"熟练使用 React" — 这句话等于没说。面经显示字节面试官必问 Fiber 架构和 Hooks 原理，建议写清楚你用 React 做了什么、解决了什么性能问题。 严重
项目经历全是课程作业 — 面经显示该岗位偏好有真实业务经验的候选人。没有实习经历的话，至少放一个有用户量的开源项目或独立作品。 严重
性能优化经验为零 — 牛客面经中 80% 提到了 Lighthouse 评分优化、首屏加载、懒加载等，你的简历一个字都没提。 缺失
TypeScript 提了但没展开 — 你说"了解 TypeScript"，但面经显示字节前端团队全面使用 TS，建议改为具体描述你用 TS 做了什么。 建议强化
💡

修改建议

1
用 STAR 法则重写项目经历：将"使用 React 开发了一个电商网站"改为"独立开发基于 React 18 的电商 SPA，通过 React.lazy + Suspense 实现路由级代码分割，首屏 LCP 从 3.2s 降至 1.1s"。
2
补充工程化实践：加入 Webpack/Vite 配置优化、CI/CD 流程搭建、单元测试覆盖率等内容。面经显示这是字节前端的加分项。
3
删除无关技能："熟练使用 Office 办公软件"对前端岗位毫无意义，浪费版面。把空间留给 Node.js、构建工具等相关技能。
